# Справочник служебных файлов 1С:Предприятие

Полное описание форматов и назначения служебных файлов платформы 1С:Предприятие 8.3.

---

## Когда использовать

Используйте этот справочник когда:
- Работаете с выгрузкой/загрузкой конфигураций
- Настраиваете автоматическое развертывание
- Создаете инсталляционные пакеты
- Работаете с хранилищем конфигураций
- Настраиваете списки информационных баз
- Создаете обновления конфигураций
- Работаете с внешними обработками и отчетами

---

## Основные типы служебных файлов

### 1. Файлы конфигураций

#### *.cf — Файл конфигурации
**Назначение:** Выгрузка конфигурации в бинарном формате для распространения или создания новых информационных баз.

**Как создать:**
```
1cv8.exe DESIGNER /F"<путь_к_ИБ>" /DumpCfg "<путь>\config.cf"
```

**Применение:**
- Распространение типовых конфигураций
- Создание новых информационных баз
- Резервное копирование конфигурации

---

#### *.cfu — Файл обновления конфигурации
**Назначение:** Файл обновления для существующей конфигурации.

**Как создать:**
```
1cv8.exe DESIGNER /F"<путь_к_ИБ>" /CreateDistributionFiles -cffile "<путь>\update.cfu"
```

**Применение:**
- Обновление типовых конфигураций
- Распространение исправлений
- Поставки обновлений клиентам

---

#### *.cfe — Файл расширения конфигурации
**Назначение:** Выгрузка расширения конфигурации.

**Как создать:**
```
1cv8.exe DESIGNER /F"<путь_к_ИБ>" /DumpConfigToFiles "<путь_к_каталогу>" -Extension <имя_расширения>
```

**Применение:**
- Распространение расширений
- Версионирование расширений
- Перенос расширений между базами

---

### 2. Файлы данных

#### *.dt — Файл выгрузки данных
**Назначение:** Полная выгрузка информационной базы (конфигурация + данные).

**Как создать:**
```
1cv8.exe DESIGNER /F"<путь_к_ИБ>" /DumpIB "<путь>\database.dt"
```

**Как загрузить:**
```
1cv8.exe DESIGNER /F"<путь_к_новой_ИБ>" /RestoreIB "<путь>\database.dt"
```

**Применение:**
- Полное резервное копирование
- Перенос базы между серверами
- Создание копий для тестирования
- Миграция с файлового варианта на клиент-сервер

---

### 3. Файлы обработок и отчетов

#### *.epf — Внешняя обработка
**Назначение:** Исполняемый файл внешней обработки.

**Создание:** Через конфигуратор или EDT
- Конфигуратор: Файл → Сохранить как → Внешняя обработка
- Программно: `ВнешняяОбработка.Записать(ИмяФайла)`

**Применение:**
- Служебные обработки для администрирования
- Инструменты для пользователей
- Временные доработки без изменения конфигурации

---

#### *.erf — Внешний отчет
**Назначение:** Исполняемый файл внешнего отчета.

**Создание:** Аналогично *.epf

**Применение:**
- Дополнительные отчеты
- Специализированные аналитические формы
- Временные отчеты

---

### 4. Файлы хранилища конфигураций

#### *.1ccr — Файл подключения к хранилищу
**Назначение:** Конфигурационный файл для подключения к хранилищу конфигурации через TCP.

**Формат файла (XML):**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<repository connectString="tcp://ИмяСервера:Порт/ИмяХранилища"/>
```

**Пример:**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<repository connectString="tcp://RepServ:1542/MainRepository"/>
```

**Применение:**
- Централизованное управление подключениями
- Автоматизация работы с хранилищем
- Развертывание на рабочих местах разработчиков

---

### 5. Файлы манифестов

#### *.mft — Файл-манифест дистрибутива
**Назначение:** Описание состава дистрибутива конфигурации для автоматической установки.

**Формат:** INI-файл в кодировке UTF-8

**Структура файла:**
```ini
Vendor=<Поставщик>
Name=<Наименование>
Version=<Версия>
AppVersion=<Версия платформы>

[<Идентификатор поставки>]
Source=<Путь к cf/cfu/dt файлу>
Catalog_ru=<Путь в каталоге>
Destination=<Целевой каталог>
```

**Пример:**
```ini
Vendor=Фирма "1С"
Name=Управление торговлей
Version=11.5.0.1
AppVersion=8.3.27

[Main]
Source=Main\1cv8.cf
Catalog_ru=Управление торговлей/Управление торговлей
Destination=1C\Trade

[Demo]
Source=Demo\1cv8.dt
Catalog_ru=Управление торговлей/Демо-база
Destination=1C\DemoTrade
```

**Параметры:**
- `Vendor` — Поставщик решения
- `Name` — Наименование конфигурации
- `Version` — Версия конфигурации
- `AppVersion` — Требуемая версия платформы
- `Source` — Путь к файлу (cf, cfu, dt)
- `Catalog_ru` — Путь в каталоге (локализация)
- `Destination` — Целевая папка установки

**Применение:**
- Автоматическая установка конфигураций
- Развертывание обновлений
- Установочные пакеты

---

### 6. Списки информационных баз

#### *.v8i — Список информационных баз
**Назначение:** Файл со списком информационных баз для автоматического добавления в список пользователя.

**Расположение по умолчанию:**
- **Windows:** `%APPDATA%\1C\1CEStart\ibases.v8i`
- **Linux:** `~/.1C/1cestart/ibases.v8i`
- **macOS:** `~/.1C/1cestart/ibases.v8i`

**Формат:** INI-файл в кодировке UTF-8

**Структура:**
```ini
[Наименование базы]
Connect=<Строка подключения>
ID=<UUID>
Folder=<Путь к папке в дереве>
OrderInList=<Порядок>
OrderInTree=<Порядок в дереве>
App=<Тип приложения>
Version=<Версия платформы>
DefaultVersion=<Признак версии по умолчанию>
ClientConnectionSpeed=<Скорость соединения>
```

**Примеры строк подключения:**

**Файловая база:**
```ini
Connect=File="D:\Bases\Accounting";
```

**Серверная база:**
```ini
Connect=Srvr="server1c";Ref="Accounting";
```

**Веб-приложение:**
```ini
Connect=ws="http://server1c/accounting/";
```

**Основные параметры:**

- `Connect` — Строка подключения (обязательный)
- `ID` — Уникальный идентификатор базы (UUID)
- `Folder` — Путь к папке в дереве списка
- `OrderInList` — Порядковый номер в списке
- `App` — Тип приложения:
  - `Auto` — Автоматически
  - `ThinClient` — Тонкий клиент
  - `ThickClient` — Толстый клиент
  - `WebClient` — Веб-клиент
- `Version` — Версия платформы (например, `8.3.27.1445`)
- `DefaultVersion` — Признак версии по умолчанию (`yes`/`no`)
- `ClientConnectionSpeed` — Скорость соединения:
  - `Normal` — Обычная
  - `Low` — Низкая

**Полный пример:**
```ini
[Бухгалтерия предприятия 3.0]
Connect=Srvr="server1c";Ref="Accounting";
ID=a1b2c3d4-e5f6-1234-5678-90abcdef1234
Folder=Рабочие базы
OrderInList=1
OrderInTree=1
App=Auto
Version=8.3.27.1445
DefaultVersion=yes
ClientConnectionSpeed=Normal
```

**Применение:**
- Централизованное управление списком баз
- Автоматическое добавление баз на рабочих местах
- Развертывание для новых пользователей
- Резервное копирование настроек

### 7. XML-выгрузки конфигураций

#### Выгрузка в файлы (каталог с XML)
**Назначение:** Выгрузка конфигурации в текстовом формате для работы с системами контроля версий.

**Как создать:**
```
1cv8.exe DESIGNER /F"<путь_к_ИБ>" /DumpConfigToFiles "<путь_к_каталогу>"
```

**Как загрузить:**
```
1cv8.exe DESIGNER /F"<путь_к_ИБ>" /LoadConfigFromFiles "<путь_к_каталогу>"
```

**Структура каталога:**
```
Configuration/
├── Configuration.xml          # Корневой файл конфигурации
├── CommonModules/             # Общие модули
│   ├── ОбщийМодуль1.xml
│   └── ОбщийМодуль1/
│       └── Ext/
│           └── Module.bsl
├── Catalogs/                  # Справочники
│   └── Номенклатура/
│       ├── Номенклатура.xml
│       └── Forms/
│           └── ФормаЭлемента/
│               ├── Form.xml
│               └── Ext/
│                   └── Form/
│                       └── Module.bsl
├── Documents/                 # Документы
├── DataProcessors/           # Обработки
├── Reports/                  # Отчеты
└── ...
```

**Применение:**
- Работа с Git, SVN и другими VCS
- Командная разработка
- Сравнение версий конфигураций
- Code review
- Автоматизация сборки

### 8. Дополнительные файлы

#### *.cfl — Файл журнала изменений
**Назначение:** Лог-файл изменений конфигурации в режиме конфигуратора.

**Расположение:** Каталог информационной базы

**Применение:**
- Отладка проблем
- Аудит изменений
- Анализ ошибок

---

#### *.lgf, *.lgp, *.lgd — Технологический журнал
**Назначение:** Детальное логирование работы платформы.

**Настройка:** Файл `logcfg.xml` в каталоге установки платформы или ИБ

**Применение:**
- Диагностика производительности
- Отладка сложных проблем
- Аудит безопасности
- Мониторинг системы

---

#### *.1CD — Файловая информационная база
**Назначение:** Основной файл файловой информационной базы.

**Расположение:** Указывается при создании базы

**Применение:**
- Однопользовательская работа
- Небольшие базы
- Тестовые базы
- Разработка

## Практические сценарии использования

### Сценарий 1: Выгрузка и загрузка конфигурации

**Выгрузка конфигурации в файл:**
```batch
"C:\Program Files\1cv8\8.3.27.1445\bin\1cv8.exe" DESIGNER ^
  /F"D:\Bases\MyBase" ^
  /N"Администратор" ^
  /DumpCfg "D:\Export\config.cf"
```

**Загрузка конфигурации из файла:**
```batch
"C:\Program Files\1cv8\8.3.27.1445\bin\1cv8.exe" DESIGNER ^
  /F"D:\Bases\MyBase" ^
  /N"Администратор" ^
  /LoadCfg "D:\Export\config.cf" ^
  /UpdateDBCfg
```

---

### Сценарий 2: Выгрузка и загрузка базы данных

**Выгрузка:**
```batch
1cv8.exe DESIGNER ^
  /F"D:\Bases\Production" ^
  /N"Администратор" /P"password" ^
  /DumpIB "D:\Backups\backup_20240316.dt"
```

**Загрузка в новую базу:**
```batch
1cv8.exe DESIGNER ^
  /F"D:\Bases\TestCopy" ^
  /N"Администратор" ^
  /RestoreIB "D:\Backups\backup_20240316.dt"
```

---

### Сценарий 3: Работа с XML-выгрузкой для Git

**Выгрузка в XML:**
```batch
1cv8.exe DESIGNER ^
  /F"D:\Bases\Development" ^
  /N"Разработчик" ^
  /DumpConfigToFiles "D:\Git\MyProject\src" ^
  -Format Hierarchical
```

**Загрузка из XML после merge:**
```batch
1cv8.exe DESIGNER ^
  /F"D:\Bases\Development" ^
  /N"Разработчик" ^
  /LoadConfigFromFiles "D:\Git\MyProject\src" ^
  /UpdateDBCfg
```

---

### Сценарий 4: Создание файла обновления

**Создание CFU:**
```batch
1cv8.exe DESIGNER ^
  /F"D:\Bases\NewVersion" ^
  /N"Администратор" ^
  /CreateDistributionFiles ^
  -cffile "D:\Updates\update_v11.5.cfu" ^
  -cfufile "D:\Updates\manifest.mft"
```

---

### Сценарий 5: Автоматическое добавление базы в список

**Создание файла базы.v8i:**
```ini

Folder=Тестовые базы
```

**Применение:** Двойной клик по файлу автоматически добавит базу в список

## Рекомендации и best practices

### Безопасность

1. **Не храните пароли в командных файлах**
   - Используйте переменные окружения
   - Запрашивайте пароль интерактивно
   - Используйте системы управления секретами

2. **Ограничьте доступ к служебным файлам**
   - .v8i файлы могут содержать строки подключения
   - .1ccr файлы указывают на хранилище
   - Используйте права доступа ОС

3. **Шифруйте резервные копии**
   - .dt файлы содержат все данные
   - Используйте шифрование при передаче
   - Храните в защищенном месте

### Производительность

1. **Оптимизируйте выгрузку/загрузку**
   - Используйте SSD для временных файлов
   - Выполняйте операции вне рабочего времени
   - Мониторьте дисковое пространство

2. **Кэшируйте статические файлы**
   - .cf файлы типовых конфигураций
   - .mft манифесты
   - .v8i списки баз

### Версионирование

1. **XML-выгрузки для Git**
   - Используйте `-Format Hierarchical`
   - Настройте .gitignore для временных файлов
   - Используйте pre-commit hooks

2. **Именование файлов**
   - Включайте версию: `config_v11.5.0.123.cf`
   - Добавляйте дату: `backup_20240316.dt`
   - Используйте понятные имена

3. **Структура каталогов**
   ```
   Project/
   ├── configs/
   │   ├── v11.5.0/
   │   │   └── config.cf
   │   └── v11.5.1/
   │       └── config.cf
   ├── updates/
   │   └── v11.5.0_to_v11.5.1/
   │       ├── update.cfu
   │       └── manifest.mft
   ├── src/               # XML-выгрузка для Git
   │   └── Configuration/
   └── backups/
       └── 2024/
           └── 03/
               └── backup_20240316.dt
   ```

## Автоматизация с PowerShell

### Скрипт резервного копирования

```powershell
# Параметры
$platform = "C:\Program Files\1cv8\8.3.27.1445\bin\1cv8.exe"
$infobase = "D:\Bases\Accounting"
$backupDir = "D:\Backups\$(Get-Date -Format 'yyyy\\MM')"
$backupFile = "backup_$(Get-Date -Format 'yyyyMMdd_HHmmss').dt"

# Создание каталога
New-Item -ItemType Directory -Force -Path $backupDir | Out-Null

# Выгрузка
$arguments = @(
    "DESIGNER"
    "/F`"$infobase`""
    "/N`"Администратор`""
    "/DumpIB `"$backupDir\$backupFile`""
)

Start-Process -FilePath $platform -ArgumentList $arguments -Wait -NoNewWindow

Write-Host "Резервная копия создана: $backupDir\$backupFile"
```

---

### Скрипт обновления из хранилища

```powershell
$platform = "C:\Program Files\1cv8\8.3.27.1445\bin\1cv8.exe"
$infobase = "D:\Bases\Development"
$repository = "D:\Settings\repository.1ccr"

$arguments = @(
    "DESIGNER"
    "/F`"$infobase`""
    "/N`"Разработчик`""
    "/ConfigurationRepositoryF `"$repository`""
    "/ConfigurationRepositoryN `"developer`""
    "/ConfigurationRepositoryUpdateCfg"
    "/UpdateDBCfg"
    "/UC DELETE-ALL-SESSIONS"
)

Start-Process -FilePath $platform -ArgumentList $arguments -Wait -NoNewWindow
```

## Частые проблемы и решения

### Проблема: "Не удается подключиться к информационной базе"

**Причины:**
- Неверная строка подключения в .v8i
- База заблокирована другим процессом
- Недостаточно прав доступа

**Решение:**
1. Проверьте строку `Connect=` в .v8i файле
2. Убедитесь, что база не открыта в другом сеансе
3. Проверьте права доступа к каталогу базы

---

### Проблема: "Несовместимая версия конфигурации"

**Причины:**
- .cf файл создан в более новой версии платформы
- Требуется обновление базы данных

**Решение:**
1. Используйте платформу версии не ниже указанной в `AppVersion` манифеста
2. Выполните обновление конфигурации БД: `/UpdateDBCfg`

---

### Проблема: "Ошибка при загрузке из .dt файла"

**Причины:**
- Поврежденный файл выгрузки
- Недостаточно места на диске
- Несовместимость версий

**Решение:**
1. Проверьте целостность .dt файла (размер, контрольная сумма)
2. Освободите место на диске (требуется ~3x размер .dt)
3. Используйте ту же или более новую версию платформы
