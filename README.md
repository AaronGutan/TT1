# ⚠️ Важные правила разработки для проекта 1С:Предприятие

> **КРИТИЧЕСКИ ВАЖНО**: Эти правила основаны на официальных стандартах разработки 1С:Предприятие и должны соблюдаться во всех проектах для обеспечения качества, поддерживаемости и соответствия стандартам платформы.

Этот репозиторий содержит **обязательные** правила и соглашения о коде для разработки на платформе 1С:Предприятие, оформленные в формате MDC (Markdown Context) для использования в Cursor IDE.

## Структура правил

Правила разделены на логические модули в каталоге `.cursor/rules/`:

- **`1c-standards.mdc`** ⭐ (КРИТИЧНО) - Основные соглашения о коде, стиль, именование, структура модулей
  - `alwaysApply: true` - применяется всегда
  - Приоритет 1 (высший)

- **`1c-queries-collections.mdc`** - Работа с запросами, циклами и универсальными коллекциями значений
  - `alwaysApply: false` - применяется при необходимости
  - Приоритет 2

- **`1c-client-server.mdc`** - Клиент-серверное взаимодействие
  - `alwaysApply: false` - применяется при необходимости
  - Приоритет 3

- **`1c-errors-security.mdc`** ⭐ (КРИТИЧНО) - Обработка ошибок, безопасность и производительность
  - `alwaysApply: true` - применяется всегда
  - Приоритет 4

## ⚠️ Важность соблюдения правил

Эти правила **обязательны** для соблюдения, так как они:
- Основаны на официальных стандартах разработки 1С:Предприятие
- Обеспечивают единообразие кода в команде
- Повышают читаемость и поддерживаемость кода
- Предотвращают типичные ошибки и проблемы производительности
- Обеспечивают безопасность и корректную обработку ошибок
- Соответствуют лучшим практикам платформы 1С

## Использование

### Для Cursor IDE

1. Скопируйте содержимое каталога `.cursor/rules/` в ваш проект
2. Правила автоматически применятся при работе с файлами `.bsl` и `.os`
3. **Рекомендуется**: Установить `alwaysApply: true` для критичных правил (строки 1 и 4)

### Настройка приоритетов

В файлах правил можно настроить:
- `alwaysApply: true/false` - применять всегда или только при необходимости
- `priority: 1-4` - приоритет применения правил
- `globs` - паттерны файлов, к которым применяются правила
